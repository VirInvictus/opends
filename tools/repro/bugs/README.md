# Bug fixtures

Each subdirectory here is one repro fixture. The harness in
[`../repro.py`](../repro.py) takes the directory name as its
positional argument; `python3 repro.py ds1-smoke` runs the
`ds1-smoke` fixture against `.games/ds1/`.

`python3 repro.py --list` enumerates all fixtures with their
target game and one-line description.

## Catalogue

| Fixture     | Game | Bug catalogue ref | What it proves |
|-------------|------|-------------------|----------------|
| `ds1-smoke` | DS1  | (smoke; not a bug) | DS1 boots, lives 25+ seconds, harness can SIGTERM cleanly. Validates the v0.1.0 harness pattern end-to-end. |
| `ds2-smoke` | DS2  | (smoke; not a bug) | DS2 boots through `imgmount` of `game.ins` for CD audio, lives 25+ seconds. Validates the DS2 path of the harness. |

The `bug catalogue ref` column will cross-link to entries in
[`../../../docs/known-bugs.md`](../../../docs/known-bugs.md)
once real-bug fixtures land (v0.3.0+).

## Adding a new fixture

See [`../README.md`](../README.md) "Adding a new bug fixture"
for the schema and the workflow. Always include this row in the
catalogue above (it's the index a maintainer reads first when
they're triaging a regression).

## The MEL audio gotcha (read before adding any DS1 / DS2
fixture that boots the engine)

Both engines abort on launch with the factory `SOUND.CFG`
because MEL (Miles Audio Library) fails the SB DSP probe. The
fix is to ship a `SOUND.CFG` generated by running
`sound_ds.exe` inside DOSBox under Wine, selecting Sound
Blaster, and capturing the file from
`~/.wine/drive_c/GOG Games/Dark Sun [2]/SOUND.CFG`. The
existing `ds1-smoke` and `ds2-smoke` fixtures each check in
their own such file. See [`../README.md`](../README.md)
"The audio gotcha" for the full story.
