# OpenDS Tools

The toolkit. Each tool is independent and shippable on its own.
Each has its own `README.md`, `VERSION`, and tagged GitHub
releases.

See [`../docs/versioning.md`](../docs/versioning.md) for the
versioning policy and [`../spec.md`](../spec.md) §7a for the
implementation-language split.

## Shipped

| Tool                                     | Lang   | Version | Purpose                                                                              |
|------------------------------------------|--------|---------|--------------------------------------------------------------------------------------|
| [`atlas`](atlas/)                        | Python | 0.1.0   | **Static-HTML site generator.** `atlas build --games-dir DIR -o OUT` drives image-extract / region-render / dialog-extract against every detected game install and produces a browsable HTML directory (file://). Three sections per game: sprite gallery, region maps inline, dialog browser. Cross-section nav threaded through every page; no JavaScript. Corpus smoke: 1685 sprites, 53 region maps, 2 dialog browsers, 92 MB site. The "open the whole game and look around" surface; item #7 of the human-friendliness sprint.|
| [`opends`](opends/)                      | Rust   | 0.1.0   | **Umbrella CLI.** Auto-dispatches by file magic. `opends inspect <file>` reads the first bytes (or filename heuristic for saves) and runs `gff-cat info` / `save-inspect` / a PNG summary; `opends tools` prints a version table of every wrapped tool; `opends extract / render / find` are thin shells over `gff-cat bulk-extract` / `region-render` / `dialog-extract --grep`. New-contributor entry point ("I have this file, what is it?"). Workspace tool discovery prefers in-tree `target/release/` over `$PATH`. Item #3 of the human-friendliness sprint.|
| [`verify-install`](verify-install/)      | Python | 0.3.0   | Check a Dark Sun install against the canonical pristine-hash manifest. v0.2.0 added `--json` + `--repair <installer.exe>` (innoextract; backs up to `__verify-install-backup/<path>`; `--dry-run` previews). **v0.3.0** adds `--rollback` (inverse of `--repair`; restores every file in the backup dir and removes it; pairs with `--dry-run`) and `--summary` (one-line plain-English status instead of the full table; for the common-case modder check). Pluralises properly; frames the next-step advice so the user doesn't have to remember which flag does what.|
| [`gff-edit`](gff-edit/)                  | Rust   | 0.6.0   | Pure-Rust GFF read/write (library `gff_edit` + CLI `gff-cat`). Read/write + bulk extract + text codec + JSON + catalogue + builder (v0.5.0 indexed-only). **v0.6.0** adds `gff-cat what <kind> <id>`: per-chunk describer that combines the kind purpose, chunk size, header-derived facts (bitmap dimensions, text preview), and a next-step pointer to the right tool. The "I have this chunk, what do I do with it?" answer in one command. Dispatches by FOURCC family (BMP → image-extract; GPL/MAS → gpl-disasm; TEXT → gff-cat dump-text; CHAR/SAVE → save-inspect; RMAP/GMAP → region-render). Segmented-type builder remains deferred (promoted only when a consumer needs it).|
| [`gpl-disasm`](gpl-disasm/)              | Rust   | 0.6.0   | GPL bytecode disassembler. 100% corpus alignment on DS1+DS2 GPL/MAS. Recursive-descent CFG with labeled branches (v0.3.x), hand-curated symbol catalogue (v0.4.0 / v0.4.2), inter-chunk callgraph (v0.4.1), lossless 7-bit packed-string decoder (v0.4.3), Deserialize impls (v0.4.4), `raw_tail` side-byte preservation (v0.4.5), and a public `render_text` API with round-trippable label / raw_tail rendering (v0.4.6). `--json` mode for downstream tools. v0.5.0 added global-variable naming via `syms/variables.toml` (`GBYTE[42 (POV_FLAGS)]` decoration); `gpl-asm` accepts the decorated form on parse. **v0.6.0** is the first ship of the human-friendliness sprint: per-chunk local-variable overlays (`syms/locals.toml` keyed by `(file, kind, chunk_id, var_kind, id)` so decoration fires only inside the named chunk; same `LBYTE[7 (loop_counter)]` render shape as globals); a stdlib-only DSO-symbol importer at `scripts/import-dso-symbols.py` (parses `.dso-online/tools/symbols.txt`, emits 100 opcode-rename proposals + 15 unmatched `Decode*` candidates + GPL/GFF engine-intrinsic markdown summaries; script never writes to `syms/` directly, every commit is hand-reviewed); plus two latent v0.5.0 bug fixes (single-chunk path now decorates globals; CLI early-out now considers variables + locals so a syms dir with only those isn't discarded). Tests: three new lib tests for locals overlay + 600 / 600 corpus round-trip unchanged.|
| [`gpl-asm`](gpl-asm/)                    | Rust   | 0.7.0   | GPL bytecode reassembler. Consumes `gpl-disasm --json` (600/600 byte-identical) or the text listing in either form. v0.3.0 added the `Editor` API for structural edits; v0.4.0 adds label-relative APIs and user-chosen label names. v0.5.0 shipped the author safety net (caret-style parse errors + a `validate()` pass wired as the default pre-encode check). v0.6.0 added `%define <name> <replacement>` and `%search-tail <hex-bytes>`. **v0.7.0** adds two real authoring features: parameterised macros (`%define name(p1, p2) <body>` with call-site expansion `name(a, b)`; arg pre-expansion against the outer defines so plain-define values flow naturally into macro args) and `@include "path/file.asm"` (textual include relative to the current file, with a canonical-path circular-include guard and an `INCLUDE_DEPTH_LIMIT = 16` circuit-breaker). Decorated-variable parse from gpl-disasm v0.5.0 also lands here. Corpus stays at 600 / 600.|
| [`save-inspect`](save-inspect/)          | Python | 0.8.0   | Dump CHARSAVE / DARKRUN / SAVE0N as JSON, diff two, OR **edit-and-write-back**. v0.1.0 - v0.7.0 shipped the read path; **v0.8.0 closes the loop with `save-edit`** (JSON-in, GFF-out; backup + dry-run) plus a pure-Python `write_gff` and a `roundtrip` regression subcommand. Encoders shipped for every existing decoder (combat / character / item DS1+DS2; PSIN/PSST/TEXT/STXT/SAVE/ETME/ETAB). Round-trip is **100% chunk-level byte-identical** across the corpus: 27/27 DS1 CHARSAVE, 98/98 DS2 CHARSAVE, 63/63 Brandon's played DARKRUN. End-to-end smoke (decode → edit CHAR-36 hp=54 to 999 → save-edit → re-decode = hp 999) confirms the first true mod workflow on the toolkit. File-level output compacts the original's 1.6 KB of pre-allocated inter-chunk gaps; engine-equivalent, smaller. Fidelity fixes for combat name padding (raw-hex preservation), DS1 item truncation trailing byte, and oversized opaque chunks (full _raw_bytes_hex alongside truncated preview).|
| [`dialog-extract`](dialog-extract/)      | Python | 0.7.0   | Pull GPL strings (NPC dialog, prompts, NPC names) from GPL/MAS chunks. v0.1-v0.6 shipped the structured JSON dialog tree with CFG-aware path tracking, callgraph-narrowed possible_writers, and distance ordering. **v0.7.0** adds two human-readable outputs: `--format transcript` (per-NPC plain-text dialog listing with speaker labels from a curated `syms/speakers.toml`; DS1 GPLDATA produces an 18349-line transcript) and `--format html` (single-file static HTML browser with embedded CSS, collapsible `<details>` per chunk, colour-coded unresolved strings; opens via `file://`). `--format json` (default) stays unchanged for back-compat. The on-ramp to the planned `atlas` static-site tool.|
| [`image-extract`](image-extract/)        | Rust   | 0.4.0   | Extract and pack Dark Sun bitmap chunks (`BMP `, `PORT`, `ICON`, `BMAP`, `OMAP`, `TILE`). v0.1.0 - v0.3.0 ship the decoder (chunk → palette-indexed PNG; decodes DS1 RLE, PLNR, PLAN; 99.95% of corpus frames decode cleanly). v0.3.0 added multi-frame export via `--frames-all` / `--spritesheet`. **v0.4.0** ships the inverse: a new `image-pack` binary that reads a palette-indexed 8-bit PNG and emits a DS1 RLE chunk. Library `encode_bitmap_rle(frames)` with greedy RLE + multi-span row emission (handles 320-pixel sprite rows whose payload exceeds the single-span 255-byte cap). 883 / 883 corpus DS1 RLE frames round-trip pixel-identical; end-to-end smoke (extract ICON 2000 → `image-pack` → `gff-cat replace` → re-extract) lands byte-identical. The toolkit's first true sprite-modding workflow.|
| [`region-render`](region-render/)        | Rust   | 0.6.0   | Render a region GFF's full visual stack: background tiles, walls, and entity sprites into a 2048x1568 palette-indexed PNG. v0.4.0 adds `--palette-preset {ds1-pink, ds1-rust, ds1-deep-red}` for one-knob DS1 palette switching. v0.5.0 lands a DSUN.EXE RE pass (`docs/dsun-exe-re.md`) that located the engine's `CMAT[id] -> CPAL[id]` per-region palette routine; default DS1 fallback now uses `CPAL:200` (engine-default) instead of `PAL :1000` (menu palette). **v0.6.0** pivots past the cycle-table wall to animate the **entity layer** via `image-extract v0.3.0`'s multi-frame decoder: `--animate-entities` emits a numbered PNG sequence stepping each ETAB-referenced sprite through its `frame_count` cycle (DS1 RGN02 max is 15 frames); `--frame-count N` overrides the auto-picked max. Frame 0 of the animated render is byte-identical to v0.5.0 single-frame output. Palette animation still queued.|
| [`repro`](repro/)                        | Shell + Python | 0.4.0 | DOSBox-Staging repro harness. Boots a per-bug fixture under `bugs/<id>/` with overlay-mounted writes (the install stays byte-identical), evaluates pass/fail by elapsed time + scratch-dir artifacts. v0.1.0 - v0.3.0 shipped the harness pattern, smoke fixtures, DOSBox stderr capture, `--play` + `--session` continuity. **v0.4.0** adds scheduled keystrokes (`[[trigger.keystrokes]]` driving `ydotool` with friendly aliases / `type:` / raw scancode pairs; daemon-thread scheduler) and video capture (`record_video = true` driving `ffmpeg -f x11grab` to `<scratch>/repro.mp4`; XWayland surface visible so GNOME-Wayland works without a portal). Both gracefully degrade when the dep is missing. README walks through one-time Fedora setup. Unblocks `opcode-fuzz v0.3.0`'s deterministic-execution loop.|

| [`opcode-fuzz`](opcode-fuzz/)            | Python | 0.3.0   | The GPL opcode-discovery harness. v0.1.0 shipped chunk-patchwork (`extract` / `pack` / `roundtrip`, 600 / 600 byte-identical). v0.2.0 added `run` (pack + DOSBox launch via repro --play --session + DARKRUN snapshot + byte-level diff). **v0.3.0** adds `boot-chunks <gff>`: drives `gpl-disasm --global-cfg --json` and surfaces chunks with zero inbound `gpl global sub` edges (engine must dispatch them directly; safest swap targets). DS1 GPLDATA: 129 boot candidates / 250 chunks; DS2: 196 / 350. `recipes/` scaffold lands as forward-looking docs; recipe-driven `fuzz` waits on a settled recipe format. Sessions inherit repro v0.4.0's input automation when ydotool is installed.|
